Network Toolkit Help

Portable technician toolkit for live workstation and network troubleshooting.

Start Here

Use Quick Diagnosis first. It creates a technician-friendly HTML report and a Computer Profile for the local machine. The repair tools should be used only after Quick Diagnosis indicates servicing or system file follow-up, unless you deliberately override the warning.

The toolkit is designed for live troubleshooting. Reports, temp tool outputs, and profiles are retained in bounded folders so you can compare recent results without filling the USB drive indefinitely.

Quick Diagnosis and Repair

Quick Diagnosis is the primary triage path. It runs diagnostic-only checks and saves a Computer Profile. DISM/SFC repair remains separate. If Quick Diagnosis does not indicate servicing follow-up, the GUI will ask for override confirmation before repair runs.

Temp Tool Outputs

Console-driven tools create timestamped output sessions under Data\TempToolOutputs. These sessions capture metadata and a transcript for troubleshooting tool usage without collecting every GUI log line. The toolkit keeps a limited recent history and removes older sessions automatically.

Chocolatey

Use Search Packages to find packages and Install Selected to install. Use Scan Installed to list local packages. Upgrade Selected and Uninstall Selected act on the selected installed package. If a package is already installed, the toolkit will show a message instead of launching a pointless install console.

Minidumps and BSOD Review

The Minidump Collector copies the selected dump into a temp output session. Point BlueScreenView or WinDbg at that copied folder so analysis does not depend on locked or protected Windows dump locations.

Robocopy Builder Tips

The current builder is intentionally balanced. Useful future enhancements would be saved presets, source/destination browse buttons, estimated action summary, and a dry-run diff view. Those would add power without making the builder much harder for junior technicians.
